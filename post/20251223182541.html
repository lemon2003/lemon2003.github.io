生成对抗网络详解学习笔记

第一部分：生成模型与生成器

1.1 为什么需要生成模型？

在前几章中，我们学习的神经网络本质上都是一个确定性的函数，提供输入x，网络就会输出一个确定的结果y。这类网络是判别模型，它们擅长进行分类、回归等判断性任务。

然而，世界上很多任务需要的是“创造性”的输出，即一个问题存在多个正确的答案。例如：

  • 视频预测：给定一段游戏视频，预测下一帧画面。在角色走到岔路口时，下一帧角色向左或向右都是合理的。
  • 聊天机器人：回答“你知道哪些童话故事？”时，答案可以是安徒生童话、格林童话等，没有唯一标准答案。
  • 绘画：根据“红眼睛”这个描述，每个人画出的角色都不一样。

    如果使用传统的判别模型进行视频预测，网络会学习训练数据中所有可能的情况（既向左又向右）。为了“讨好”所有数据，它会输出一个模糊的、取平均后的结果，导致画面出现残影，角色轮廓不清。这正是因为判别模型只会输出一个“最可能”的答案，无法处理多模态输出。

1.2 生成器：输出一个分布

为了解决这个问题，我们引入了生成模型。其核心是生成器，它不再是输出一个确定值，而是输出一个复杂的概率分布，包含了所有可能的合理答案。

生成器的关键设计是引入随机性。它的输入除了原始数据x，还有一个从简单随机分布（如高斯分布、均匀分布）中采样得到的噪声变量z。

  • 公式：y = G(x, z)
  • 效果：对于同一个输入x，由于每次采样的z不同，生成器的输出y也会不同。这样，生成器就能覆盖多种可能性。

    在小精灵游戏的例子中，生成器可以学到：当z采样到某个值时，角色向左转；当z采样到另一个值时，角色向右转。这样就完美解决了多答案问题。

    生成器的应用：无限制生成，即不需要条件x，只输入噪声z来生成数据。例如，从高斯分布中采样一个低维向量z，输入生成器，就能输出一张动漫人脸。不同的z会生成不同的人脸。

第二部分：生成对抗网络的基本原理

2.1 判别器：真伪鉴定专家

只有生成器还不够，我们需要一个机制来指导它生成逼真的内容。这就是判别器。判别器也是一个神经网络，它的任务是判断输入的数据是“真实的”还是“生成器伪造的”。

  • 输入：一张图片（或其他数据）
  • 输出：一个标量分数，分数越高代表图片越真实。

2.2 GAN的训练过程：“警匪博弈”

GAN的精髓在于生成器G和判别器D之间的对抗性训练，这就像一场“警匪博弈”：

  • 生成器G（匪）：目标是生成尽可能逼真的假数据，以骗过判别器。
  • 判别器D（警）：目标是练就火眼金睛，准确分辨真实数据和生成数据。

训练过程是分步交替进行的：

第一步：训练判别器D（固定生成器G）

  1. 从真实数据集中采样一批真实图片，标签为1。
  2. 从噪声分布中采样z，输入生成器G得到一批生成图片，标签为0。
  3. 将这两批图片混合，训练判别器D进行二分类（或回归），目标是让D对真实图片打高分，对生成图片打低分。

    第二步：训练生成器G（固定判别器D）
  4. 从噪声分布中采样z，输入生成器G得到生成图片。
  5. 将生成图片输入已经固定的判别器D，得到分数。
  6. 训练生成器G，目标是让D给生成图片打的分数越高越好。通过这个反馈，G学习如何改进以生成更逼真的图片。

    循环往复：不断重复第一步和第二步。判别器D和生成器G在对抗中不断进化，最终达到一个平衡点：生成器G能生成足以乱真的数据，而判别器D难以分辨，真假概率各为50%。

2.3 GAN的强大能力

通过这种对抗训练，GAN展现出惊人的能力：

  • 生成逼真图像：可以生成不存在的高清人脸、动漫角色等。
  • 连续性创造：对噪声向量z进行插值，生成器可以产生连续变化的图像（如从严肃到微笑，从左看向右看），这表明生成器学习到的是一个有意义的“概念空间”，而非简单的记忆。

第三部分：GAN的理论基础与WGAN的改进

3.1 理论目标：缩小分布差异

从理论上看，GAN的训练目标是让生成器产生的数据分布，尽可能接近真实数据的分布。我们需要一个衡量两个分布差异的指标。

原始GAN使用JS散度作为衡量标准。通过一番推导，判别器D的目标函数最大值与JS散度相关。训练GAN的过程就是在解一个最小最大优化问题：生成器G试图最小化JS散度，而判别器D试图最大化JS散度（通过最大化其目标函数来实现）。

3.2 传统GAN的训练难题：JS散度的缺陷

然而，传统GAN训练极其困难，被称为“痛苦的开盲盒”，其根源在于JS散度的缺陷：

  1. 高维空间中的“细线”：真实图片分布和生成图片分布，在高维空间中就像是两条几乎不可能相交的“细线”。
  2. JS散度“失灵”：当两个分布完全没有重叠时（这在训练初期是常态），JS散度会恒等于一个常数log2。这意味着，无论是离很远还是靠近了一点，JS散度这个“尺子”都显示同样的值，无法提供有效的训练信号。
  3. 梯度消失：由于JS散度恒定，判别器D会很快达到接近100%的准确率，导致其提供给生成器G的梯度（反馈信号）变得非常微弱甚至为零。G因此无法学习，训练陷入停滞。

3.3 WGAN的救赎：Wasserstein距离

为了解决JS散度的问题，Wasserstein GAN提出使用Wasserstein距离（也称推土机距离）作为新的衡量标准。

直观理解：想象有两个土堆P和Q。Wasserstein距离是将土堆P挪成土堆Q的形状，所需的最小“工作量”（移动的土方量×移动距离）。

核心优势：即使两个分布和没有重叠，只要它们相互靠近，Wasserstein距离就会连续、平滑地减小。这为生成器提供了稳定、有意义的训练梯度。

  • 判别器的新角色：此时判别器更像一个“评论家”，给图片打连续分数，而不是简单判断真假。
  • 关键约束：1-Lipschitz连续性：为了防止判别器“作弊”（给真图打正无穷分，假图打负无穷分），必须要求判别器函数足够“平滑”。即输入图片的微小变化，不会引起输出分数的剧烈波动。

如何实现平滑？—— 梯度惩罚
最主流的方法是梯度惩罚。在真实图片和生成图片的连线上随机采样点，然后在判别器的损失函数中增加一项，强制这些点处的梯度范数接近1，从而保证判别器的平滑性。

第四部分：GAN的评估与进阶模型

4.1 如何评估GAN的性能？

评估生成器的好坏是一个重要课题。早期只能靠人眼观察，主观且不稳定。现在有客观的指标：

  1. Inception Score

    • 质量：将生成图片输入预训练的图像分类网络，输出概率分布越集中，说明图片质量越高、内容越明确。
    • 多样性：将大量生成图片的概率分布平均，平均分布越平坦，说明生成多样性越好。
    • 缺点：将质量和多样性分开评估，有时会矛盾。

  2. Fréchet Inception Distance（FID）—— 当前金标准

    • 步骤：
      1. 将真实图片和生成图片分别输入Inception网络。
      2. 提取中间层的特征向量。
      3. 假设这两组特征向量各自服从一个多元高斯分布。
      4. 计算这两个高斯分布之间的Fréchet距离。
    • 意义：FID值越小，说明生成图片的特征分布与真实图片越接近，即同时兼顾了质量和多样性。

4.2 GAN的常见问题：模式崩溃

模式崩溃是GAN训练中的一个典型失败情况。生成器“偷懒”，找到了判别器的一个盲点，只生成少数几种能骗过判别器的图片，完全放弃了数据的多样性。

4.3 条件生成对抗网络

我们希望控制生成器的输出，这就是条件GAN。我们给生成器和判别器都提供一个额外的条件信息x（如一段文字、一张草图）。

  • 生成器：y = G(x, z)。根据条件x和噪声z生成图片y。
  • 判别器：同时看条件x和图片y，判断“图片y不仅真实，而且与条件x匹配”。

    应用：文字生成图像、图像修复（如着色）、图像转换（如草图变实景Pix2Pix）等。

4.4 CycleGAN：无配对数据的风格迁移

条件GAN需要成对的数据（如一张草图对应一张实景图），但这类数据很难获得。CycleGAN解决了无配对数据的域转换问题。

核心思想：循环一致性
训练两个生成器：

  • 生成器G：将X域图片（如真人照）转换为Y域风格（如动漫）。
  • 生成器F：将Y域图片转换回X域风格。

循环一致性损失：要求一张X域图片x，经过G变成Y域图片G(x)，再经过F变回X域图片F(G(x))后，应该与原始图片x尽可能相似（即F(G(x)) ≈ x）。反之亦然。

这个约束强迫生成器保留输入图片的核心内容，只改变风格。CycleGAN成功应用于季节转换、物体变形、照片风格化等任务。

总结：生成对抗网络通过生成器和判别器的对抗博弈，让机器学会了“创造”。从解决传统模型的多答案问题，到克服训练不稳定的WGAN，再到实现可控生成的条件GAN和无监督学习的CycleGAN，这一系列技术构成了当前AIGC爆发的核心基础。理解其原理、挑战和解决方案，是深入人工智能领域的关键一步。
