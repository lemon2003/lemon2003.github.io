阿里云购买域名并配置GitHub Pages博客：从零到一的完整指南

在当今数字时代，拥有一个个人博客或项目展示网站已成为开发者和创作者的标配。GitHub Pages提供免费的静态网站托管服务，而阿里云则提供稳定可靠的域名注册服务。本文将手把手教你如何将两者结合，打造一个专业、免费的个人博客。

为什么选择GitHub Pages + 阿里云域名？

GitHub Pages的优势：

  • 完全免费，无需服务器费用

  • 支持自定义域名

  • 自动HTTPS加密

  • 与Git版本控制无缝集成

  • 支持Jekyll等静态网站生成器
    阿里云域名的优势：

  • 国内访问速度快

  • 管理界面友好

  • 价格实惠（.top等小众域名新人首年1元）

  • 无需备案即可绑定GitHub Pages（托管在海外服务器）

第一步：在阿里云购买域名

1.1 注册阿里云账号

访问阿里云官网，点击右上角”注册”按钮。如果你已有淘宝或支付宝账号，可以直接登录。

1.2 查询并选择域名

在阿里云首页搜索框中输入你想要的域名，点击”查询域名”。系统会显示该域名是否可注册以及价格信息。

1.3 完成购买流程

  1. 将可用域名加入购物车
  2. 选择注册年限
  3. 创建信息模板并完成实名认证
  4. 选择支付方式完成支付
    重要提示：根据中国法规，所有域名必须完成实名认证。个人用户需上传身份证正反面照片，企业用户需上传营业执照。未完成实名认证的域名将无法正常解析。

第二步：创建GitHub Pages博客

2.1 创建GitHub仓库

  1. 登录GitHub账号
  2. 点击右上角”+”号，选择”New repository”
  3. 仓库名格式：username.github.io（username替换为你的GitHub用户名）
  4. 选择”Public”，跟自己，你不想让别人访问，就设置成私有
  5. 点击”Create repository”

2.2 启用GitHub Pages

  1. 进入新建的仓库
  2. 点击”Settings” → “Pages”
  3. 在”Source”部分，选择分支（通常为main或master）
  4. 点击”Save”
  5. 等待几分钟，访问https://username.github.io查看效果（有时候会访问不了，这是GitHub特性）

2.3 添加博客内容

你可以使用以下方式创建博客内容：

  • 简单HTML/CSS：直接上传HTML文件
  • Jekyll静态网站：GitHub Pages原生支持
  • Hexo/Hugo：通过GitHub Actions自动构建
  • React/Vue项目：构建后上传dist文件夹

第三步：配置自定义域名

3.1 阿里云DNS解析配置

登录阿里云控制台，进入”域名控制台” → 选择你的域名 → “解析设置”。


添加A记录（根域名解析）：

需要添加4条A记录，分别指向GitHub Pages的4个IP地址：

记录类型 主机记录 记录值
A @ 185.199.108.153
A @ 185.199.109.153
A @ 185.199.110.153
A @ 185.199.111.153

添加CNAME记录（www子域名）：

记录类型 主机记录 记录值
CNAME www username.github.io

将username替换为你的GitHub用户名。

3.2 GitHub仓库配置

  1. 在仓库根目录创建名为CNAME的文件（无后缀）
  2. 文件内容只写你的域名，如：example.com
  3. 提交并推送更改

或者通过GitHub设置：

  1. 进入仓库”Settings” → “Pages”
  2. 在”Custom domain”输入框中填写你的域名
  3. 点击”Save”
  4. 勾选”Enforce HTTPS”（一般阿里云那边解析完成后，这边很快）

4.2 等待生效

DNS更改通常需要10分钟到48小时才能全球生效。你可以通过以下方式测试：

  1. 访问你的域名查看是否显示GitHub Pages内容
  2. 检查浏览器地址栏是否有HTTPS锁标志
  3. 使用在线DNS检测工具如dnschecker.org

常见问题与解决方案

Q1：域名无法访问

  • 原因：DNS记录未生效或配置错误
  • 解决：检查DNS记录是否正确，等待24-48小时传播

Q2：HTTPS证书错误

  • 原因：混合内容（HTTP/HTTPS混用）或DNS未完全生效
  • 解决：检查网页资源链接，确保全部使用HTTPS协议

Q3：www子域名无法访问

  • 原因：CNAME文件内容错误或DNS记录指向错误
  • 解决：确保CNAME文件仅包含域名，DNS记录正确指向username.github.io

Q4：GitHub Pages显示404

  • 原因：仓库设置错误或文件路径问题
  • 解决：检查GitHub Pages设置，确保选择了正确的分支和文件夹

进阶配置

同时支持根域名和www子域名

GitHub Pages会自动在两者间创建重定向。配置example.com时，访问www.example.com会自动跳转；反之亦然。

自定义404页面

在仓库根目录创建404.html文件，GitHub Pages会自动使用它作为404错误页面。

性能优化建议

  1. CDN加速：结合Cloudflare免费套餐启用全球CDN
  2. 资源压缩：使用gzip或TinyPNG压缩静态资源
  3. 缓存策略：通过_config.yml配置静态资源缓存周期
  4. 图片优化：使用WebP格式替代JPEG/PNG
  5. 代码精简：移除未使用的CSS/JavaScript

总结

通过阿里云购买域名并配置GitHub Pages博客，你可以在零服务器成本的情况下拥有一个专业、安全的个人网站。整个过程主要包括三个核心步骤：

  1. 购买域名：在阿里云选择并注册心仪的域名，完成实名认证
  2. 创建博客：在GitHub创建username.github.io仓库，启用GitHub Pages
  3. 绑定域名：配置DNS解析，在GitHub仓库添加CNAME文件

这种组合方案特别适合个人开发者、学生、技术博主和初创项目。它不仅成本低廉，而且维护简单，让你可以专注于内容创作而非服务器管理。

最后提醒：虽然GitHub Pages托管在海外服务器，绑定域名无需备案，但如果你未来考虑使用国内服务器，仍需完成ICP备案。建议定期检查域名续费时间，避免因过期导致网站无法访问。

现在，开始打造你的个人品牌吧！一个专业的域名加上精心维护的技术博客，将成为你职业生涯中宝贵的数字资产。
