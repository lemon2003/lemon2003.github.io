Supabase配置 首先需要在Supabase创建项目和表，这是基础。

1. 新建Supabase项目

  • 字段1：id（类型UUID，设为默认主键）；

  • 字段3：phone（类型varchar，存储手机号）；

三、踩坑记录(这个仅供提醒，完整代码部分是错误都解决的)：

这些问题一定要注意！ 过程中遇到了很多报错，总结几个关键坑点：

1. 跨域问题：No 'Access-Control-Allow-Origin'

原因：Supabase的CORS配置未允许前端域名。
解决：初始化Supabase时添加options.credentials = 'omit'，关闭跨域凭证检查。

2. API密钥错误：No API key found in request

原因：密钥格式不对（复制了别名而非完整密钥）。
解决：在Supabase控制台的Settings → API中，复制完整的Publishable Key（以eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9开头）。

3. 变量未初始化：Cannot access 'supabase' before initialization

原因：代码执行顺序错误，事件监听在supabase初始化之前。
解决：先初始化supabase变量，再绑定表单的事件监听。

4. 字段名不匹配：数据无法写入表

原因：代码中字段名和Supabase表字段名不一致（比如表是中文，代码用了英文）。
解决：确保代码中insert/select的字段名与表结构完全一致。

四、部署：

GitHub Pages上线 把代码保存为index.html，推送到GitHub仓库，开启GitHub Pages服务：

  • 仓库→Settings→Pages→选择分支（如main）→保存；
  • 等待部署完成，访问分配的域名（如https://你的用户名.github.io/仓库名/）即可使用。

五、总结

用前端直连Supabase做简单的注册登录，优势是无需后端、部署简单，适合快速搭建原型。核心是处理好跨域、密钥、字段名这几个关键点，避开文中的坑就能顺利跑通~
